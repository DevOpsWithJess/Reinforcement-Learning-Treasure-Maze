## Project Overview

For this project, I built a reinforcement learning–based pirate intelligent agent using a Jupyter Notebook environment. The goal was to train an agent to navigate the environment successfully using reward-based learning. The starter code provided the environment structure and some framework for running episodes, but I was responsible for implementing and tuning the learning components.

I created and adjusted the training loop, implemented the epsilon-greedy strategy, monitored reward trends across episodes, and evaluated how the agent improved over time. I also analyzed performance results and made adjustments to exploration rates and learning behavior to improve stability and convergence.


## What Do Computer Scientists Do and Why Does It Matter?

This project helped me see that computer scientists don’t just “write code.” We design systems that learn, adapt, and make decisions. In this project, I wasn’t telling the agent exactly what to do step by step, I was designing a system that adjusts its behavior based on the results it experiences. That shift really changes how you think about building software.
Reinforcement learning has real-world applications in robotics, autonomous systems, finance, healthcare decision support, and more. Understanding how models learn from rewards, how exploration balances with exploitation, and how performance is evaluated is critical when building systems that may influence real-world outcomes.


## How I Approach a Problem as a Computer Scientist

Throughout this course, I learned to approach problems more systematically. Instead of trying to “make it work,” I focused on understanding the environment, defining the objective clearly, testing incrementally, and analyzing results before making adjustments.

When training the agent, I paid attention to patterns in the reward progression, shifts in exploration rate, and whether the model was truly learning or just getting lucky. I learned that iteration and evaluation are just as important as the initial implementation.


## Ethical Responsibilities to End Users and Organizations

Working with reinforcement learning made it clear that intelligent systems can influence behavior in meaningful ways. That creates responsibility.

As a computer scientist, I have an ethical obligation to design systems that are transparent, secure, and reliable. Models should not be deployed without proper validation. Data should be handled responsibly, and outcomes should be monitored to prevent bias or unintended consequences.

Even though this pirate agent exists in a simulated environment, the same principles apply to real-world AI systems. Power requires accountability, and thoughtful design matters just as much as technical performance.
